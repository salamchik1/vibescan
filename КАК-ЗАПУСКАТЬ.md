# ▶️ Как запускать VibeScan (шпаргалка)

Это твой проект-сканер. Он состоит из **двух частей, которые работают одновременно**:

| Часть | Что делает | Адрес |
|---|---|---|
| **Сканер** (`apps/scanner`) | мотор: ходит на чужой сайт, ищет дыры | http://localhost:8787 |
| **Веб** (`apps/web`) | то, что ты видишь: страница, форма, отчёт | http://localhost:3000 |

Ты заходишь на **http://localhost:3000**, вставляешь адрес сайта — веб тихо зовёт сканер, сканер всё проверяет и возвращает отчёт 🔴🟡🟢.

## 🌐 Чтобы работал ПУБЛИЧНЫЙ сайт (vibescan-web.vercel.app)

Сайт живёт на Vercel, а сканер — на твоём ПК. Чтобы сайт мог звать твой сканер,
запусти **онлайн-агента** — он сам поднимает сканер, открывает туннель и
**публикует текущий адрес в Supabase**. Vercel читает этот адрес на лету, поэтому
**в Vercel больше ничего менять и редеплоить не нужно**.

**Запуск:** двойной клик по **`Сканер-онлайн.bat`** (или в терминале `npm run online`).
Держи окно открытым — пока оно открыто и ПК включён, публичный сайт работает.
Закрыл / выключил ПК → сайт честно пишет «сканер офлайн».

### Настройка один раз (на стороне Vercel)
1. В Vercel → проект `vibescan-web` → Settings → Environment Variables:
   - **удали** `SCANNER_URL` (чтобы сайт брал адрес из Supabase автоматически);
   - `SCANNER_SHARED_SECRET` = тот же секрет, что в твоём `.env`;
   - убедись, что заданы `NEXT_PUBLIC_SUPABASE_URL` и `SUPABASE_SERVICE_ROLE_KEY`.
2. В Supabase → SQL Editor выполни свежий [`supabase/schema.sql`](supabase/schema.sql)
   (он добавит таблицу `scanner_endpoint`; повторный запуск безопасен).
3. Redeploy на Vercel — **последний раз**. Дальше адрес обновляется сам.

### Чтобы стартовало само при включении ПК
1. `Win+R` → вставь `shell:startup` → Enter (откроется папка автозагрузки).
2. ПКМ по **`Сканер-онлайн-тихо.vbs`** → «Создать ярлык», и положи ярлык в эту папку.
3. Теперь при каждом входе в Windows агент стартует тихо, без окон.

## 🚀 Самый простой способ — в один клик (без чёрных окон)

В корне проекта лежат два файла:

- **`Запустить-VibeScan.vbs`** — запустить;
- **`Остановить-VibeScan.vbs`** — остановить.

1. **Дважды кликни** по **`Запустить-VibeScan.vbs`**.
2. **Никаких командных окон не появится** — серверы (сканер и веб) запускаются тихо, в фоне.
3. Через несколько секунд **сам откроется браузер** на http://localhost:3000.

Можно спокойно закрывать вкладку и открывать снова: серверы продолжают работать в фоне. Кликнул по `Запустить-VibeScan.vbs` ещё раз — он просто **снова откроет вкладку** (дубли не создаются).

**Чтобы выключить серверы:** дважды кликни по **`Остановить-VibeScan.vbs`** (тоже без окон).

> **Первый запуск** (или после удаления `apps\web\.next`) ненадолго покажет одно окно «first-time setup» — это идёт сборка/установка. Оно само закроется, и дальше запуски снова тихие.
>
> **После правок кода** удали папку `apps\web\.next` и запусти `Запустить-VibeScan.vbs` снова — он пересоберёт свежую версию (продакшн-режим, стабильный и без ошибок кэша).

## 🧑‍💻 Способ для разработки (вручную, два терминала)

Если правишь код — удобнее видеть логи в терминале. Открой **два** терминала в папке проекта:

```powershell
# Терминал 1 — сканер (мотор)
npm run dev:scanner      # поднимется на http://localhost:8787

# Терминал 2 — веб (витрина)
npm run dev:web          # поднимется на http://localhost:3000
```

Оба запущены в режиме **watch**: правишь файл → сервер сам перезапускается. Потом открой http://localhost:3000 вручную.

## ✅ Как проверить, что сканер живой

Открой в браузере http://localhost:8787/health — должен ответить:

```json
{"ok":true,"version":"0.1.0"}
```

Если ответил — мотор работает.

## 🔍 Как просканировать сайт

1. Открой http://localhost:3000
2. Вставь **публичный URL** (например, своего тестового приложения на Lovable/Bolt)
3. Поставь галочку «я имею право сканировать этот сайт»
4. Жми **Scan** → через 30–90 сек увидишь отчёт 🔴🟡🟢

> ⚠️ Сканируй только **свои** сайты или те, на которые есть разрешение. Сканер специально **блокирует** localhost и внутренние/приватные адреса (SSRF-защита) — чужой `localhost` просканировать нельзя, это by design.

## 🛠 Если что-то не запускается

| Симптом | Что сделать |
|---|---|
| Браузер пишет «не удаётся подключиться» к :3000 | Веб ещё не успел подняться — подожди 10–15 сек и обнови страницу. |
| Сканер не отвечает на `/health` | Останови (`Остановить-VibeScan.vbs`) и запусти заново. Если не помогло — посмотри логи через «Способ для разработки» ниже. |
| `npm` не найден | Не установлен Node.js. Поставь Node ≥ 20 с https://nodejs.org , перезапусти. |
| Порт занят (`EADDRINUSE`) | Уже запущена старая копия — дважды кликни `Остановить-VibeScan.vbs`, затем запусти заново. |
| Совсем чистая установка | В терминале: `npm install`, затем `npx playwright install chromium`. |
